# 执行型改造提示词模板

你现在是本项目的核心程序员，请按照以下要求对项目进行端到端改造、审查、测试与整理。禁止只给建议，必须直接修改代码并验证结果。

## 目标
- 保持“功能覆盖广 + 快速演进”的节奏。
- 同时补齐安全与工程治理短板：
  - 证书校验
  - 降级策略
  - 生产开关统一
  - 异常策略一致性

## 必做事项
1. 统一运行环境判定
- 统一 APP_ENV/MAINCOIN_ENV/POUW_ENV/MAINCOIN_PRODUCTION 的判定逻辑。
- 禁止各模块各自定义“生产模式”。

2. 安全失败关闭（Fail-Closed）
- 生产环境中，TLS 初始化失败必须拒绝启动。
- 生产环境中，TLS 客户端禁止 CERT_NONE；必须配置 CA 并启用证书校验。

3. 启动前安全基线检查
- 启动前检查关键配置并在生产环境中阻断：
  - 管理员密钥是否固定
  - 本地请求是否强制鉴权
  - 是否允许请求头覆盖认证主体
  - 是否配置 CA 证书

4. 高风险执行路径收敛
- 生产环境默认禁用进程内 CUSTOM_CODE 执行。
- 仅允许容器化执行路径，或通过显式环境变量放开并告警。

5. 回归测试与 CI
- 新增安全基线测试：环境判定、证书校验、CUSTOM_CODE 门禁。
- 将测试加入安全回归工作流，防止后续回退。

## 输出要求
- 必须输出：
  1) 修改了哪些文件
  2) 每个文件改了什么、为什么
  3) 测试执行结果
  4) 仍存风险与下一步计划

## 约束
- 最小改动原则，不改无关代码。
- 不得引入明文密钥、后门开关或不安全默认值。
- 对暂时无法彻底解决的问题，必须写明原因与替代方案。
