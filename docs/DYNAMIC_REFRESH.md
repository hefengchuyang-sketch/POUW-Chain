# POUW Chain 动态刷新功能说明

## ✅ 已实现的动态刷新功能

### 1. 移除数据缓存
- **问题**: 之前使用 `@st.cache_resource` 缓存 MiningScheduler，导致数据不更新
- **解决**: 移除缓存装饰器，每次都获取最新数据
- **影响**: 挖矿页面的矿工信息、任务状态都会实时更新

### 2. 实时余额显示
所有页面的余额查询都改为实时从 RPC 获取，不使用缓存：

#### 仪表盘 (Dashboard)
- ✅ 每次打开都重新查询余额
- ✅ 添加刷新按钮和时间戳显示
- ✅ 显示最后刷新时间

#### 挖矿页面 (Mining)
- ✅ 显示实时板块币余额卡片
- ✅ 矿工详情显示链上实时余额（非本地统计）
- ✅ 每次都从 RPC 查询最新数据

#### 兑换页面 (Exchange)
- ✅ 添加刷新按钮
- ✅ 显示刷新时间戳
- ✅ 余额实时更新

### 3. 手动刷新按钮
每个页面都添加了 🔄 刷新按钮：
- 仪表盘右上角
- 挖矿页面标题旁
- 兑换页面标题旁

### 4. 自动刷新选项
在侧边栏底部添加了"🔄 自动刷新 (5秒)"选项：
- 勾选后每5秒自动刷新整个页面
- 确保所有数据都是最新的
- 适合监控挖矿收益

## 🎯 使用说明

### 查看实时余额
1. 登录钱包后，在仪表盘即可看到所有余额
2. 每个数字旁边显示刷新时间 `⏰ HH:MM:SS`
3. 点击 🔄 按钮立即刷新

### 监控挖矿收益
1. 进入"⛏️ 挖矿中心"页面
2. 查看顶部的板块币余额卡片
3. 展开矿机详情可看到该板块的实时余额
4. 开启侧边栏的"自动刷新"可每5秒更新

### 兑换板块币
1. 进入"💱 实时汇率"页面
2. 可看到所有板块币余额和预览
3. 点击"一键兑换所有板块币为 MAIN"
4. 兑换成功后自动刷新显示新余额

## 📊 数据流向

```
挖矿客户端提交区块
    ↓
node_launcher.py 的 mining_submitBlock
    ↓
调用 sector_ledger.mint_block_reward()
    ↓
铸造板块币到矿工地址 (写入 sector_coins.db)
    ↓
Web UI 调用 account_getBalance RPC
    ↓
从 sector_ledger 读取实时余额
    ↓
显示在界面上
```

## 📝 技术细节

### 余额查询流程
```python
# 每次都实时查询，不缓存
balances_resp = rpc.call("account_getBalance", {"address": wallet_address})

# RPC 调用 node_launcher.py 的 account_get_balance
# → 查询 sector_ledger.get_balance()
# → 从 SQLite 数据库读取最新余额
```

### 自动刷新实现
```python
auto_refresh = st.checkbox("🔄 自动刷新 (5秒)", value=False)
if auto_refresh:
    import time
    time.sleep(5)
    st.rerun()  # 重新运行整个页面
```

## ⚡ 性能说明

- 余额查询：SQLite 读取，毫秒级响应
- 刷新频率：推荐5秒（自动）或手动刷新
- 网络开销：仅 RPC 调用，本地通信
- 数据库负载：轻量级查询，无压力

## 🐛 故障排查

### 余额不更新？
1. 确认节点正在运行
2. 检查 RPC 端口 8545 是否开放
3. 查看节点日志是否有错误
4. 点击刷新按钮或开启自动刷新

### 挖矿收益为0？
1. 确认挖矿客户端正在运行
2. 查看区块是否成功提交（终端输出）
3. 检查 sector_coins.db 是否有数据
4. 重启节点应用修复（mint_block_reward）
