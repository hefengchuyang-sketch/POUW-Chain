# POUW Chain 合约系统说明

> **版本**: 2.0.0  
> **最后更新**: 2026-03-08

---

## 概述

POUW Chain 的合约系统是围绕**算力市场**设计的一套自动化协议层，涵盖算力定价、预算锁定、任务结算、期货预定和争议仲裁。与传统智能合约（如 Solidity）不同，POUW 的合约是**协议内置的确定性逻辑**，不支持用户自定义编程，确保了经济模型的安全性和可预测性。

### 核心模块

| 模块 | 实现文件 | 功能 |
|------|---------|------|
| **动态定价引擎** | `core/dynamic_pricing.py` | 供需自动定价、多策略支持 |
| **预算锁定合约** | `core/dynamic_pricing.py` → `BudgetLockManager` | 下单时锁定预算，完成后多退少补 |
| **结算引擎** | `core/dynamic_pricing.py` → `SettlementEngine` | 自动计费、分账、链上记录 |
| **算力期货合约** | `core/compute_futures.py` → `FuturesContractManager` | 预定未来算力、价格锁定 |
| **算力经济合约** | `core/compute_economy.py` → `ComputeContract` | 多类型合约（现货/日/周/月/年） |
| **加密任务结算** | `core/encrypted_task.py` → `TimeBillingEngine` | 加密任务计费与结算 |
| **合约审计** | `core/smart_contract_audit.py` → `ContractAuditor` | 代码安全审计与漏洞检测 |

## 1. 动态定价机制

### 定价公式

$$\text{单价} = \text{基础价格} \times \text{供需系数} \times \text{时段系数} \times \text{策略系数}$$

### 基础价格（MAIN/GPU·小时）

| GPU 型号 | 基础价格 | | GPU 型号 | 基础价格 |
|---------|---------|---|---------|---------|
| CPU | 1.0 | | RTX 4070 | 12.0 |
| RTX 3060 | 5.0 | | RTX 4080 | 18.0 |
| RTX 3070 | 7.0 | | RTX 4090 | 25.0 |
| RTX 3080 | 10.0 | | A100 | 40.0 |
| RTX 3090 | 15.0 | | H100 | 60.0 |
| | | | H200 | 80.0 |

### 供需系数（0.5x ~ 3.0x）

系统根据全网矿工供给和任务需求的比率，使用对数函数平滑计算供需系数：

- 供过于求 → 系数降至 0.5x → 价格便宜
- 供不应求 → 系数升至 3.0x → 价格上涨
- 每 5 分钟重新计算一次

### 时段系数

| 时段 | 时间范围 | 系数 |
|------|---------|------|
| 高峰 | 9:00—12:00, 14:00—18:00 | ×1.3 |
| 正常 | 其他时段 | ×1.0 |
| 低谷 | 0:00—6:00 | ×0.7 |

### 定价策略

| 策略 | 系数 | 最长等待 | 适合场景 |
|------|------|---------|---------|
| **立即执行** (`IMMEDIATE`) | ×1.5 | 1 分钟 | 紧急任务 |
| **标准** (`STANDARD`) | ×1.0 | 10 分钟 | 日常使用 |
| **经济** (`ECONOMY`) | ×0.8 | 1 小时 | 不急的任务 |
| **夜间** (`NIGHT`) | ×0.6 | 24 小时 | 可等到凌晨 |
| **弹性** (`FLEX`) | ×0.9 | 2 小时 | 系统智能调度 |

## 2. 预算锁定合约

### 工作原理

用户提交算力任务时，系统按**最坏情况**自动锁定预算，确保任务一定能完成：

```
用户提交任务
    ↓
系统计算最坏情况价格 = 最高市场系数 × 高峰时段 × 策略系数 × 1.1 安全边际
    ↓
从用户余额中锁定该金额
    ↓
任务执行，按实际分钟计费
    ↓
任务完成 → 结算实际消费 → 退还多锁定的部分
```

### 关键参数

| 参数 | 值 | 说明 |
|------|------|------|
| 锁定过期时间 | 预估时长 × 2 | 超时自动释放 |
| 安全边际 | ×1.1 | 防止价格波动导致不足 |
| 最低计费 | 0.1 MAIN | 任何任务的最低收费 |
| 退款方式 | 自动 | 结算后立即退回用户余额 |

### 预算保护承诺

- 你**永远不会超额消费**——实际费用永远 ≤ 锁定金额
- 锁定金额是最坏情况估算，实际消费通常远低于锁定金额
- 任务取消时**全额退款**

### RPC 接口

| 方法 | 功能 |
|------|------|
| `budget_deposit` | 充值预算（需解锁钱包） |
| `budget_getBalance` | 查询预算余额 |

## 3. 自动结算引擎

### 结算流程

```
任务完成
    ↓
结算引擎计算实际费用（按时间片累加）
    ↓
从锁定预算中扣除实际费用
    ↓
退还多余锁定金额到用户账户
    ↓
支付矿工（结算额的 90%）
平台运维（5%）
DAO 国库（5%）
    ↓
生成结算记录（含链上哈希）
```

### 分账比例

| 去向 | 比例 | 说明 |
|------|------|------|
| 矿工 | 90% | 算力提供者 |
| 平台运维 | 5% | 网络维护 |
| DAO 国库 | 5% | 公共基金 |

### 结算记录

每笔结算生成不可篡改的链上记录，包含：

| 字段 | 说明 |
|------|------|
| `settlement_id` | 结算唯一标识 |
| `task_id` | 关联任务 |
| `user_id` / `miner_id` | 买卖双方 |
| `locked_budget` | 原始锁定金额 |
| `actual_cost` | 实际消费 |
| `refund_amount` | 退款金额 |
| `settlement_hash` | 链上哈希（SHA-256） |
| `time_slices` | 分时段计费明细 |

### RPC 接口

| 方法 | 功能 |
|------|------|
| `settlement_getRecord` | 查询结算记录 |
| `settlement_getDetailedBill` | 详细分时段账单 |

## 4. 算力期货合约

### 设计目标

允许用户**提前预定未来的算力资源**，锁定价格，避免市场价格波动风险。矿工也可以通过期货合约提前锁定订单，获得稳定收入来源。

### 合约类型

| 类型 | 说明 | 最短期限 |
|------|------|---------|
| **现货** (`SPOT`) | 即时交付 | 无 |
| **期货** (`FUTURES`) | 预定未来时段算力 | 1 小时 |
| **预定** (`RESERVATION`) | 资源预留 | 1 小时 |
| **订阅** (`SUBSCRIPTION`) | 长期稳定供给 | 1 天 |

### 合约生命周期

```
创建 (PENDING) → 双方缴纳保证金 → 生效 (ACTIVE) → 交割 (DELIVERING) → 完成 (COMPLETED)
                                        ↓                    ↓
                                    取消 (CANCELLED)    违约 (DEFAULTED)
                                                            ↓
                                                      争议 (DISPUTED)
```

### 保证金机制

| 参数 | 值 |
|------|------|
| 默认保证金率 | 20%（合约总价值的 20%） |
| 双方均需缴纳 | 用户保证金 + 矿工保证金 |
| 锁定期 | 到合约期满 |
| 激活条件 | 双方保证金到账后合约自动生效 |

保证金保护双方利益：
- 用户违约 → 扣除用户保证金赔偿矿工
- 矿工违约 → 扣除矿工保证金赔偿用户

### 违约惩罚规则

| 违约时机 | 惩罚率（保证金百分比） |
|---------|---------------------|
| 24 小时前取消 | 5% |
| 12 小时前取消 | 10% |
| 1 小时前取消 | 20% |
| 开始后取消 | 50% |
| 不履约 | 30% |
| 质量问题 | 20% |
| 最低惩罚 | 1.0 MAIN |

### 交割验证

交割记录包含：

| 字段 | 说明 |
|------|------|
| 实际时长 | 交付的 GPU 小时数 |
| 平均利用率 | GPU 实际使用率 |
| 在线率 | 交割期间的 uptime |
| 质量评分 | 综合质量分（0-1.0） |
| 验证状态 | 是否经过独立验证 |

### 约束

| 参数 | 值 |
|------|------|
| 最长预定 | 30 天 |
| 最短预定 | 1 小时 |
| 交割缓冲 | 合约结束后 1 小时内完成交割 |

## 5. 算力经济合约

### 多类型合约

除了期货合约，系统还支持更灵活的经济合约类型：

| 合约类型 | 计费方式 | 适合场景 |
|---------|---------|---------|
| **现货** (`SPOT`) | 即时市场价 | 临时算力需求 |
| **按时** (`HOURLY`) | 按小时计费 | 短期测试 |
| **日合约** (`DAILY`) | 按天计费 | 日常使用 |
| **周合约** (`WEEKLY`) | 按周计费 | 中期项目 |
| **月合约** (`MONTHLY`) | 按月计费 | 生产环境 |
| **季合约** (`QUARTERLY`) | 按季计费 | 长期合作 |
| **年合约** (`ANNUAL`) | 按年计费 | 战略合作 |
| **期权** (`OPTION`) | 期权定价 | 对冲价格风险 |

### 期权合约

算力期权允许用户支付期权费锁定未来以特定价格购买算力的权利：

| 参数 | 说明 |
|------|------|
| 执行价格（Strike Price） | 约定的算力单价 |
| 期权费（Premium） | 购买权利的费用（不可退还） |
| 到期时间 | 期权有效期限 |
| 看涨/看跌 | 买权（看涨）或卖权（看跌） |

用户可以选择在到期前行权（按执行价买入算力），也可以放弃行权（仅损失期权费）。

## 6. 加密任务计费

### 时间计费引擎

加密任务采用时间计费模型：

$$\text{费用} = \text{时长} \times \text{基础费率} \times \text{GPU乘数} \times \text{时段乘数}$$

### GPU 乘数

| GPU 类型 | 乘数 |
|---------|------|
| H100 | ×5.0 |
| A100 | ×3.0 |
| RTX 4090 | ×2.0 |
| RTX 4080 | ×1.5 |
| RTX 3090 | ×1.2 |
| RTX 3080 | ×1.0 |
| CPU | ×0.3 |

### 计费流程

```
用户创建加密任务 → 预估费用 → 锁定预算
       ↓
任务加密分发给矿工链
       ↓
每个节点执行计算 → 记录执行时间 → 计算节点费用
       ↓
任务完成 → 汇总所有节点费用
       ↓
实际费用 = Σ(各节点费用)
退款 = 锁定预算 - 实际费用
```

### 完整性保障

每个节点的执行结果通过以下方式验证：

- **数据哈希**：加密前计算 SHA-256 哈希，解密后验证一致性
- **执行证明**：基于结果哈希 + 节点 ID 生成的证明签名
- **链式加密**：A 节点的输出加密后传给 B 节点，确保数据流不可篡改

## 7. 合约审计系统

### 审计流程

```
提交合约代码
    ↓
VulnerabilityDetector 自动扫描 6 类漏洞
    ↓
生成安全评分（满分 100）
    ↓
通过条件: 无 CRITICAL/HIGH 漏洞 且 评分 ≥ 70
    ↓
生成审计报告
```

### 检测的漏洞类型

| 漏洞类型 | 严重性 | 检测内容 |
|---------|--------|---------|
| **重入攻击** | HIGH | 外部调用前的状态更新检查 |
| **整数溢出** | MEDIUM | 无保护的算术运算 |
| **访问控制** | HIGH | 弱权限验证模式 |
| **时间戳依赖** | LOW | 区块时间戳滥用 |
| **未检查调用** | MEDIUM | 低级调用返回值检查 |
| **拒绝服务** | MEDIUM | 无界循环和 gas 耗尽风险 |

### 评分权重

| 严重性 | 扣分 |
|--------|------|
| CRITICAL | -25 |
| HIGH | -15 |
| MEDIUM | -8 |
| LOW | -3 |
| INFO | -1 |

### 合约状态

| 状态 | 说明 |
|------|------|
| `DRAFT` | 草稿，未审计 |
| `AUDITING` | 审计中 |
| `APPROVED` | 审计通过 |
| `DEPLOYED` | 已部署 |
| `ACTIVE` | 运行中 |
| `PAUSED` | 暂停 |
| `TERMINATED` | 已终止 |

## 8. 争议仲裁

### 争议触发条件

- 任务结果质量不达标
- 矿工未按合约交付
- 结算金额有异议
- 任务执行中出现故障

### 仲裁流程

```
一方发起争议 (OPEN)
       ↓
系统指派仲裁人 (UNDER_REVIEW)
       ↓
收集双方证据 → 仲裁裁决
       ↓
执行裁决 (RESOLVED)  或  升级处理 (ESCALATED)
```

### 争议记录

| 字段 | 说明 |
|------|------|
| `dispute_id` | 争议唯一标识 |
| `settlement_id` | 关联结算 |
| `initiator` / `respondent` | 争议双方 |
| `reason` | 争议原因 |
| `evidence` | 证据列表 |
| `amount_disputed` | 争议金额 |
| `arbitrator` | 仲裁人 |
| `ruling` | 裁决结果 |

## 9. 区块奖励分配合约

### 分配规则

每个区块的奖励按固定比例自动分配：

```
区块奖励（50.0 板块币）
├── 97%（48.5）→ 出块矿工
└──  3%（ 1.5）→ DAO 财库（MAIN_TREASURY）
```

此分配在区块产出时由共识层自动执行，不需要也不允许人工干预。

### 链上交易手续费分配

每笔链上交易收取 1% 手续费，硬编码分配：

| 去向 | 比例 | 机制 |
|------|------|------|
| 永久销毁 | 0.5% | 通缩 |
| 出块矿工 | 0.3% | 激励打包 |
| 协议费用池 | 0.2% | 多签管理 |

### DAO 财库

| 参数 | 值 |
|------|------|
| 资金来源 | 3% 区块奖励 + 5% 算力市场费 + 0.2% 手续费 |
| 多签要求 | 3/5 多签确认 |
| 支出审批 | 需 DAO 投票通过 |
| 链上透明 | 所有收支可追踪 |

## 安全设计原则

1. **不可超额消费**：预算锁定机制保证用户余额安全
2. **自动结算**：消除人工干预风险
3. **双向保证金**：期货合约要求买卖双方都缴纳保证金
4. **链上记录**：所有结算生成不可篡改的链上哈希
5. **分时计费**：按实际使用的分钟数计费，精确透明
6. **退款保障**：多锁定的预算自动退还，取消任务全额退款
7. **争议仲裁**：提供结构化的争议解决机制
