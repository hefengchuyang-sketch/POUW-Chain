# POUW Multi-Sector Chain 边界声明

本文档定义了 POUW Multi-Sector Chain 系统各模块之间的边界和职责划分，以确保架构清晰、职责明确。

## 2. 模块边界定义

### 2.1 Web UI 层 (web_ui_v2.py)

**职责**:
- 用户界面渲染
- 用户输入处理
- 会话状态管理 (仅前端状态)
- 调用 RPC 服务获取数据

**边界约束**:
- ❌ 禁止直接操作数据库
- ❌ 禁止直接修改区块链状态
- ❌ 禁止存储敏感数据
- ✅ 只能通过 RPC 与后端通信
- ✅ 只负责数据展示和用户交互

**依赖**:
- RPC 客户端
- UserDatabase (仅用于本地模式)

---

### 2.2 RPC 服务层 (rpc_service.py)

**职责**:
- 接收和解析 JSON-RPC 请求
- 参数验证
- 路由到对应的服务方法
- 格式化响应
- 错误处理

**边界约束**:
- ❌ 禁止包含业务逻辑
- ❌ 禁止直接操作存储
- ✅ 只做请求转发和响应格式化
- ✅ 验证请求格式和基本参数

**依赖**:
- Mempool
- TransactionStore
- BlockchainState
- P2P Network

---

### 2.3 用户管理模块 (user_db.py)

**职责**:
- 用户注册和登录
- 钱包创建和管理
- 密钥加密存储
- 会话令牌管理

**边界约束**:
- ❌ 禁止处理交易逻辑
- ❌ 禁止访问区块链数据
- ✅ 只管理用户相关数据
- ✅ 密码不可逆存储
- ✅ 私钥加密存储

**数据所有权**:
- users 表
- wallets 表
- sessions 表

---

### 2.4 交易系统 (transaction.py)

**职责**:
- 交易创建和验证
- Mempool 管理
- 交易持久化
- 双见证记录

**边界约束**:
- ❌ 禁止修改区块链状态
- ❌ 禁止处理用户身份
- ✅ 只处理交易生命周期
- ✅ 验证交易格式和签名

**数据所有权**:
- transactions 表
- mempool 表
- witnesses 表

---

### 2.5 区块链核心 (blockchain.py)

**职责**:
- 区块存储和查询
- 链状态管理
- 分叉处理
- 区块验证

**边界约束**:
- ❌ 禁止处理交易验证细节
- ❌ 禁止处理共识逻辑
- ✅ 只管理区块数据
- ✅ 维护链的一致性

**数据所有权**:
- blocks 表
- chain_tips 表

---

### 2.6 算力市场 (compute_market.py)

**职责**:
- 算力订单管理
- 节点注册和调度
- 订单匹配

**边界约束**:
- ❌ 禁止处理支付逻辑
- ❌ 禁止直接操作区块链
- ✅ 只管理算力资源分配

**数据所有权**:
- compute_orders 表
- compute_nodes 表

---

### 2.7 交易所/国库 (exchange_treasury.py)

**职责**:
- 买卖订单撮合
- 国库资金管理
- 汇率计算

**边界约束**:
- ❌ 禁止发起链上交易
- ❌ 禁止修改用户余额
- ✅ 只做订单撮合和记录

**数据所有权**:
- exchange_orders 表
- treasury_ledger 表

---

### 2.8 治理模块 (governance_enhanced.py)

**职责**:
- 质押管理
- 提案创建和投票
- 时间锁执行

**边界约束**:
- ❌ 禁止直接执行系统参数修改
- ✅ 只记录治理决策
- ✅ 通过回调触发执行

**数据所有权**:
- stakes 表
- proposals 表
- votes 表

---

### 2.9 PoUW 共识 (pouw_consensus.py)

**职责**:
- PoUW 证明验证
- 区块构建
- 难度调整
- 奖励计算

**边界约束**:
- ❌ 禁止直接网络通信
- ❌ 禁止用户身份处理
- ✅ 只处理共识相关逻辑

---

### 2.10 P2P 网络 (p2p_network.py, p2p_enhanced.py)

**职责**:
- 节点发现
- 消息路由
- 数据传播
- 心跳维护

**边界约束**:
- ❌ 禁止处理业务数据内容
- ✅ 只负责消息传输
- ✅ 保证消息签名验证

## 3. 数据流边界

### 3.1 用户创建交易流程

```
[Web UI] 
    │ 用户输入交易信息
    ▼
[RPC: tx_send]
    │ 验证请求格式
    ▼
[TransactionBuilder]
    │ 构建交易对象
    ▼
[Mempool.add]
    │ 验证并添加到内存池
    ▼
[P2P Network]
    │ 广播到网络
    ▼
[其他节点 Mempool]
```

### 3.2 区块生产流程

```
[UnifiedBlockBuilder]
    │ 选择共识类型
    ▼
[Mempool.get_pending]
    │ 获取待打包交易
    ▼
[PoUW/PoW Mining]
    │ 执行挖矿逻辑
    ▼
[BlockchainState.add_block]
    │ 添加到本地链
    ▼
[P2P Network]
    │ 广播区块
    ▼
[其他节点验证]
```

## 4. 安全边界

### 4.1 密钥隔离

| 层级 | 访问权限 |
|------|---------|
| Web UI | 无私钥访问 |
| RPC 服务 | 无私钥访问 |
| UserDatabase | 加密存储私钥 |
| 签名操作 | 需要用户密码解密 |

### 4.2 权限边界

| 操作 | 需要的权限 |
|------|-----------|
| 查询余额 | 无 |
| 发送交易 | 私钥签名 |
| 创建提案 | 质押权益 |
| 投票 | 质押权益 |
| 执行提案 | 时间锁到期 |

## 6. 错误处理边界

### 6.1 各层错误处理

| 层级 | 错误处理方式 |
|------|-------------|
| Web UI | 显示用户友好消息 |
| RPC | 返回标准错误码 |
| 业务逻辑 | 抛出特定异常 |
| 存储 | 事务回滚 |

### 6.2 错误传播规则

- 存储层错误 → 业务层捕获 → 返回失败状态
- 业务层错误 → RPC 层转换 → 返回错误码
- 网络错误 → 重试或标记离线

## 9. 修改边界规则

### 9.1 允许的修改

- 模块内部实现优化
- 新增 RPC 方法
- 新增数据表 (不影响已有表)

### 9.2 需要评审的修改

- 跨模块数据访问
- API 变更
- 数据格式变更

### 9.3 禁止的修改

- 破坏模块隔离
- 直接访问其他模块的私有数据
- 绕过 RPC 层直接操作
