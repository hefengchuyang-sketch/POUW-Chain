# POUW Chain 去中心化路线图

## 当前状态（开发/测试阶段）

当前实现是**简化的开发版本**，存在以下中心化问题：

### ❌ 问题清单

| 问题 | 当前实现 | 风险 |
|------|----------|------|
| 单节点 | node_launcher.py 单实例 | 单点故障 |
| 本地数据库 | SQLite 文件存储 | 数据不同步 |
| 用户数据库 | UserDatabase 存用户信息 | 隐私泄露风险 |
| 无 P2P | 无节点发现/同步 | 无法分布式 |
| 单点共识 | 矿工直接出块无验证 | 可被篡改 |

### ✅ 已正确实现

| 组件 | 实现方式 | 说明 |
|------|----------|------|
| 私钥管理 | keystore.py 本地加密 | 用户自持私钥 |
| 钱包生成 | 本地 HD 钱包 | 离线生成 |
| 交易签名 | 本地签名 | 私钥不离开本地 |

## 去中心化改造方案

### 阶段 1: 用户数据去中心化

**问题**: `UserDatabase` 存储用户名、邮箱等敏感信息

**解决方案**: 

```
当前（中心化）:
┌─────────────┐
│ UserDatabase│  ← 存储所有用户信息
│ (SQLite)    │
└─────────────┘

改进（去中心化）:
┌─────────────┐     ┌─────────────┐
│ 用户本地    │     │ 区块链      │
│ - 私钥      │     │ - 地址      │
│ - 个人资料  │     │ - 余额      │
│ - 加密数据  │     │ - 交易记录  │
└─────────────┘     └─────────────┘
      ↓                    ↓
   用户保管            公开可验证
```

**具体改动**:
1. 移除 `UserDatabase` 中的敏感字段（邮箱、用户名）
2. 用户身份 = 钱包地址（公钥派生）
3. 个人资料存储在本地加密文件，可选上传 IPFS

**协议**:
- 使用 libp2p 或自定义 TCP 协议
- 消息类型: `NEW_BLOCK`, `GET_BLOCKS`, `NEW_TX`, `PEER_LIST`
- 区块传播: Gossip 协议

### 阶段 3: 共识机制完善

**问题**: 矿工直接出块，无多节点验证

**解决方案** (PoUW 共识):

```
当前:
矿工 → 直接出块 → 存储

改进:
矿工提交 → 多验证节点验证 → 达成共识 → 出块

┌─────────┐
│ 矿工 A  │──┐
└─────────┘  │    ┌─────────────┐
             ├───→│ 共识投票    │───→ 区块确认
┌─────────┐  │    │ (2/3 验证)  │
│ 矿工 B  │──┤    └─────────────┘
└─────────┘  │
             │
┌─────────┐  │
│ 矿工 C  │──┘
└─────────┘
```

**PoUW 双验证**:
1. 计算结果哈希一致性（冗余验证）
2. 区块有效性投票（2/3 共识）

### 阶段 4: 分布式存储

**问题**: SQLite 本地存储

**解决方案**:

| 数据类型 | 存储方式 |
|----------|----------|
| 区块数据 | 每个节点完整副本 |
| 交易历史 | 每个节点完整副本 |
| 用户资料 | IPFS/Arweave (可选) |
| 计算任务 | IPFS CID 引用 |

### 1. 用户数据最小化

```python
# 移除 UserDatabase 中的敏感字段

class MinimalUserData:
    address: str      # 钱包地址（公钥派生）
```

